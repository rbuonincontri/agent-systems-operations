# Contributing

This repository documents operational practices for running AI agent systems in production.

The goal is to capture practices that emerge from real operational experience rather than theoretical design advice.

Contributions are welcome, particularly from engineers operating agent systems in real environments.

## Types of Contributions

The following contributions are especially valuable:

### 1. Documenting a Failure Mode

Many operational practices originate from production incidents.

If you have encountered a failure case in an agent system, consider documenting it.

A useful failure report typically includes:

- the operational context
- what changed
- what went wrong
- how the issue was discovered
- what operational practice would have prevented it

These reports help ground the repository in real-world experience.

---

### 2. Proposing a New Operational Practice

If you have developed an operational approach that improves reliability or control, you may propose a new practice.

A typical practice document should include:

- the problem the practice addresses
- the operational risks involved
- the proposed practice
- examples or references where possible

Practices should focus on **operational reliability** rather than framework-specific techniques.

---

### 3. Adding Research References

Relevant sources include:

- research papers
- engineering blog posts
- public incident reports
- conference talks

References should relate directly to agent system behavior or operational management.

---

## Contribution Process

1. Fork the repository.
2. Create a branch for your contribution.
3. Submit a pull request describing the proposed change.

Maintainers review pull requests before merging.

The goal is to keep the repository focused on **clear operational practices**.

---

## Scope Guidelines

This repository is not intended to be:

- a list of agent frameworks
- a catalog of tools
- general advice about AI application development

Instead, it focuses on practices required to operate agent systems reliably in production environments.

Contributions that improve clarity, document operational experience, or expand the set of practices are encouraged.
